1. What we collect

On the main ombra website, we do not intentionally collect personal data such as:

  • names
  • email addresses
  • account data
  • uploaded files on our servers
  • analytics or advertising tracking identifiers
  • profiles built from your activity

We also do not intentionally collect or store your IP address ourselves in a user database, analytics platform, or tracking system for the main site.

That said, like almost any website, some technical request data may still be handled by the infrastructure that delivers the site when your browser requests pages and assets. We do not use that for tracking, profiling, or marketing.

2. Local processing

Many ombra tools process files and text locally in your browser. In those cases, your content is not uploaded to our servers during normal use.

3. Local browser storage

Some parts of the site store information locally on your device using browser storage such as localStorage, sessionStorage, and browser cache.

We do not use ad cookies, cross-site tracking cookies, or third-party marketing pixels on the main site.

This may include:

  • appearance preferences
  • whether help prompts or notices were dismissed
  • temporary navigation state
  • tool-specific local data such as notes, calendar data, favorites, or cached settings
  • offline cache data provided through the service worker

This data stays in your browser unless you clear it.

4. Offline support and caching

ombra uses a service worker and browser caching so the site loads faster and some tools can work offline.

Cached files may include:

  • site pages
  • scripts
  • styles
  • tool pages
  • other static assets

5. Third-party services and external requests

The main site does not run analytics, but some parts of ombra load resources or send requests to third-party services.

When your browser loads fonts, downloads, CDN assets, or tool-specific API results, those providers may see technical request data as part of normal web delivery. That can include your IP address, user-agent, timestamps, and requested resources. That happens at the network level, not because ombra is running its own analytics or selling user data.

These may include:

  • Google Fonts for typography
  • GitHub for Ombra Utility Pro download links
  • CDN providers such as jsDelivr and cdnjs for certain JavaScript libraries
  • Tool-specific APIs used only when a tool requires outside data, such as currency rates, IP lookup, WHOIS/RDAP, or DNS queries

When you use those tools or resources, the relevant third party may receive technical request data such as:

  • IP address
  • browser or user-agent information
  • request time
  • requested URL or query

Those requests are made only to provide the feature you selected.

6. Files and content you use in tools

If a tool is designed for local processing, your files stay on your device unless the tool clearly depends on an external API.

If a tool needs an outside service, that is part of how the feature works.

7. No advertising tracking

We do not use:

  • ad trackers
  • behavioral profiling
  • marketing pixels
  • third-party analytics platforms on the main site
  • session replay tools
  • fingerprinting scripts intended to identify you across sites

8. Your control

You can reduce or remove stored local data by:

  • clearing browser storage
  • clearing site data or cache in your browser
  • using the site’s built-in data reset options where available

9. Changes

If this privacy policy changes, the updated version will be posted on this page with a revised date.

10. Contact

If you have privacy questions, contact ombrasupport@proton.me.

If needed later, this policy can be expanded with hosting-specific details like access logs, retention, and infrastructure handling.
